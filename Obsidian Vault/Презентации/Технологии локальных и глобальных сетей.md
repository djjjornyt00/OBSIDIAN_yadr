# Ethernet (IEEE 802.3) — основа современных ЛВС
**Уровни OSI**: Канальный + Физический 

**`Основные характеристики`**

| Параметр       | Значение                            |
| -------------- | ----------------------------------- |
| Метод доступа  | CSMA/CD (обнаружение коллизий)      |
| Топология      | Логическая шина / Физическая звезда |
| Среда передачи | Витая пара (UTP/STP), оптоволокно   |
| Дуплекс        | Full-duplex (с коммутаторами)       |

## Стандарты Ethernet
| **Стандарт**  | **Скорость** | **Кабель** | **Дальность** |
| ------------- | ------------ | ---------- | ------------- |
| 10Base-T      | 10 Мбит/с    | UTP Cat3   | 100 м         |
| 100Base-TX    | 100 Мбит/с   | UTP Cat5   | 100 м         |
| 1000Base-T    | 1 Гбит/с     | UTP Cat5e  | 100 м         |
| 10GBase-T     | 10 Гбит/с    | UTP Cat6a  | 100 м         |
| 100GBase-SR10 | 100 Гбит/с   | Оптика MMF | 100 м         |
### Структура кадра Ethernet (IEEE 802.3)
Кадр Ethernet (IEEE 802.3) — формат данных канального уровня для передачи по сети.

| Preamble | SFD    | MAC назначения | MAC источника | Type/Length | Данные (Payload) | FCS    |
| -------- | ------ | -------------- | ------------- | ----------- | ---------------- | ------ |
| 7 байт   | 1 байт | 6 байт         | 6 байт        | 2 байт      | 46-1500 байт     | 4 байт |
- **Preamble+SFD** — синхронизация
- **MAC-адреса** — 48 бит каждый
- **Type/Length** — протокол (0x0800 = IPv4) или длина
- **Payload** — данные (MTU = 1500 байт)
- **FCS** — CRC-32 для контроля ошибок

# Token Ring (IEEE 802.5) — устаревшая технология
Разработчик: IBM, 1980-е
## Принцип работы
1. Свободный маркер циркулирует по кольцу
2. Станция захватывает маркер → передаёт данные
3. Получатель копирует данные, помечает кадр
4. Отправитель удаляет кадр, выпускает новый маркер
## Характеристики
| **Параметр**  | *Значение*                                  |
| ------------- | ------------------------------------------- |
| Скорость      | 4 или 16 Мбит/с                             |
| Метод доступа | Token passing (детерминированный)           |
| Топология     | Физическая звезда (MAU), логическое кольцо  |
| Преимущество  | Нет коллизий, гарантированное время доступа |
# FDDI (Fiber Distributed Data Interface)
**Разработчик**: ANSI, 1980-е — для магистральных сетей
## Характеристики
| **Параметр**  | *Значение*                     |
| ------------- | ------------------------------ |
| Скорость      | 100 Мбит/с                     |
| Среда         | Оптоволокно (многомод/одномод) |
| Дальность     | До 200 км (одномод)            |
| Топология     | Двойное кольцо (dual ring)     |
| Метод доступа | Передача маркера               |
## Отказоустойчивость
- **Двойное кольцо**:
  Первичное кольцо → основная передача
  Вторичное кольцо → резерв (обратное направление)
-  При обрыве одного кольца:
   Автоматическое объединение в одно кольцо
# Беспроводные сети: Wi-Fi (IEEE 802.11)
**Частоты**: 2.4 ГГц, 5 ГГц, 6 ГГц (Wi-Fi 6E)  
**Метод доступа**: CSMA/CA (Collision Avoidance)
## Механизмы предотвращения коллизий
- Слушание среды перед передачей (Carrier Sense)
- Случайная задержка (backoff)
- RTS/CTS — запрос/разрешение передачи
- ACK — подтверждение получения

# Межсетевое взаимодействие (Internetworking)
Объединение разнородных сетей через маршрутизаторы в единую инфраструктуру.
## Основные принципы

**Маршрутизация** — Выбор оптимального пути (RIP, OSPF, BGP)

**Адресация** — Глобально уникальные IP-адреса

**Инкапсуляция** — Упаковка пакетов одного протокола в кадры другого

**Фрагментация** — Разбиение пакетов под разный MTU

**Единый IP** — Работает поверх любых канальных технологий
## Роль маршрутизатора
**Маршрутизатор** (Router) — устройство сетевого уровня (L3), соединяющее несколько сетей и пересылающие пакеты между ними на основе IP-адресов.
### Функции

- Маршрутизация пакетов — выбор пути по таблице маршрутизации

- Объединение сетей — связывает LAN, WAN, интернет

- Фильтрация трафика — ACL (Access Control Lists), firewall

- NAT/PAT — трансляция адресов (частные ↔ публичные) 

- Broadcast-изоляция — не пропускает broadcast между сетями

### Протоколы маршрутизации
|Тип|Протокол|Характеристика|
|---|---|---|
|**IGP** (внутренние)|RIP|Дистанционно-векторный, ≤15 хопов|
||OSPF|Состояния каналов, быстрая сходимость|
|**EGP** (внешние)|BGP|Протокол интернета, маршрутизация между AS|
