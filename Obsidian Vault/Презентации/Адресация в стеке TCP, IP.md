| Тип адреса   | Уровень OSI | Назначение                           | Пример            |
| ------------ | ----------- | ------------------------------------ | ----------------- |
| MAC-адрес    | Канальный   | Физическая идентификация адаптера    | 00:1A:2B:3C:4D:5E |
| IP-адрес     | Сетевой     | Логическая адресация и маршрутизация | 192.168.1.100     |
| Доменное имя | Прикладной  | Человеко-читаемое имя узла           | www.example.com   |
# MAC-адреса
- Формат: 48 бит (6 байт), шестнадцатеричная запись
- Структура:
  Первые 3 байта → OUI (код производителя)
  Последние 3 байта → серийный номер устройства

==Важно==: Работает только в пределах одной локальной сети; маршрутизаторы не передают MAC-адреса между сетями

# IP-адреса
## IPv4

- 32 бита, запись: 4 десятичных октета через точку
- Пример: `192.168.1.100`
- Всего адресов: ~4.3 млрд

## IPv6

- 128 бит, запись: 8 групп по 4 hex-цифры
- Пример: `2001:db8:85a3::8a2e:370:7334`
- Адресов: 340 ундециллионов

## Подсети и маски

Маска подсети определяет **границу между сетью и узлом**.

*Форматы маски:*
-  Десятичный: 255.255.255.0
-  Префиксный (CIDR): /24
-  Двоичный: 11111111.11111111.11111111.00000000
###### Пример расчета

*IP: 192.168.1.100/24*
*→ Сеть: 192.168.1.0*
*→ Хост: 100*
*→ Broadcast: 192.168.1.255*
*→ Доступно узлов: 2⁸ − 2 = 254*
**ФОРМУЛА:**
Количество узлов = 2^(32 − префикс) − 2 (минус адрес сети и broadcast)

| --------------- | -------------------------------- | ------------------------------- | ---------------------- |
| **Статический** | Ручная настройка администратором | Полный контроль                 | Трудоёмко              |
| **DHCP**        | Автоматическая выдача сервером   | Автоматизация, масштабируемость | Зависимость от сервера |

## Процесс DHCP (DORA)

1. DISCOVER — клиент ищет сервер (broadcast)
2. OFFER — сервер предлагает адрес
3. REQUEST — клиент подтверждает выбор
4. ACK — сервер выдаёт адрес в аренду (lease time)

##  ARP (Address Resolution Protocol)
Назначение: Преобразование IP → MAC для доставки кадров в локальной сети.

**Алгоритм работы:**

1. Узел знает IP получателя, но не знает MAC
2. Отправляет ARP-запрос (broadcast): «Кто имеет IP 192.168.1.100?»
3. Владелец IP отвечает (unicast) со своим MAC-адресом
4. Соответствие сохраняется в ARP-кэше (~2 минуты)
5. Далее кадр отправляется напрямую по MAC

**Команды работы с кэшем**:

- Windows: arp -a (просмотр), arp -d (очистка)
- Linux/macOS: arp -n, ip neigh, arp -d

## DNS (Domain Name System)
**Назначение**: Преобразование доменных имён ↔ IP-адресов.

Структура имени: `www.example.co`
`www` — поддомен/хост
`example` — домен второго уровня
`com` — домен верхнего уровня (TLD)
`.` — корневой домен (неявно)

### Процесс разрешения имени

Клиент → Локальный DNS → Корневой сервер → TLD-сервер (.com) → Авторитативный сервер (example.com) → Возврат IP → Кэширование (TTL)
