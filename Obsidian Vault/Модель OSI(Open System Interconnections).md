==OSI== — справочная система, что определяет как компьютеры взаимодействуют друг с другом *по сети*
В ней есть 7 уровней, каждый из которых взаимодействует с соседними слоями. Каждый имеет собственный домен. Один слой получает данные от предыдущего, одновременно передавая данные уже следующему уровню. 

### Инкапсуляция 
Это взаимодействие уровней.
При **отправке** каждый уровень **добавляет заголовок**(header), иногда и хвост, к данным → *инкапсуляция*.
*Декапсуляция* — при получении уровень снимает свою оболочку
# Структура модели OSI

- 7. **Прикладной** уровень (application layer)  → HTTP, FTP, SMTP, DNS
- 6. Уровень **представления** (presentation layer)  →  ASCII, EBCDI, SSL, форматы файлов (GIF, MP4)
- 5. **Сеансовый** уровень (session layer)  →  RPC, PAP, L2TP
- 4. **Транспортный** уровень (transport layer)  → TCP, UDP, порты
- 3. **Сетевой** уровень (network layer)  →  IP (IPv4, IPv6), маршрутизатор
- 2. **Канальный** уровень (data link layer)  →  Ethernet, MAC-адреса, сетевые коммутаторы
- 1. **Физический** уровень (physical layer)  → Кабели(оптоволокно), разъемы(RJ45)радиоволны

# Прикладной уровень
#application_layer
**Прикладной уровень** — место, где работает большинство программистов. Это то, с чем мы взаимодействуем напрямую. Браузер, почта, мессенджер.
[**Основная задача** :]([OSI модели: 7 уровней сетевой модели простым языком](https://cloud.ru/blog/osi-modeli-7-urovnej-setevoj-modeli-prostym-yazykom)) 
- обеспечить доступ к сетевым службам
- представить данные в удобном виде
Прикладной уровень именно предоставляет интерфейс для пользователя, мы не наблюдаем внутренней кухни - битов, пакетов, а только кнопки, чаты, страницы.

**Единица данных — Сообщение**

# Уровень представления
#presentation_layer
[**Уровень представления**]([Представительский уровень — Википедия](https://ru.wikipedia.org/wiki/%D0%9F%D1%80%D0%B5%D0%B4%D1%81%D1%82%D0%B0%D0%B2%D0%B8%D1%82%D0%B5%D0%BB%D1%8C%D1%81%D0%BA%D0%B8%D0%B9_%D1%83%D1%80%D0%BE%D0%B2%D0%B5%D0%BD%D1%8C)) — данный уровень отвечает за преобразование протоколов и шифрование/дешифрование данных.
**Единица данных — Сообщение**
У данного уровня есть **3 основных функции**.
## Функции

**Шифрование и дешифрование**
С помощью системы TLS(Transport Layer Secuirity) *шифрует* наши данные, а также отвечает за *расшифровку* запросов, поступающих с других серверов, для использования на прикладном уровне.
Например, передача пароля принимающему компьютеру.

**Сериализация и десериализация**
"Перевод". Данные переводятся в форматы, понятные для приложения.
[Процесс сериализации]([Сериализация — Википедия](https://ru.wikipedia.org/wiki/%D0%A1%D0%B5%D1%80%D0%B8%D0%B0%D0%BB%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F)) это преобразование объекта из оперативной памяти в последовательность байтов или другой формат, пригодный для хранения и передачи.
Десериализация соответственно обратный процесс — восстановление объекта из этого формата обратно в рабочий вид.

**Сжатие**
Данный процесс нужен для того, чтобы запросы отправлялись быстрее → ведь чем меньше битов, тем быстрее. Но с условием — сжатие без потерь.

> Чаще всего сейчас данные функции выполняются на прикладном уровне.

# Транспортный уровень
#transport_layer
Транспортный уровень обеспечивает связь между узлами.
Здесь используется обычно 2 самых популярных протокола.

**Единица данных — Сегмент/Датаграмма**.
## TCP Transmission Control Protocol
**Надежный, НО медленный**
*С установлением соединения*

Один из основных протоколов. Он используется поверх IP для обеспечения надежности передачи пакетов.
Данный протокол решает многие проблемы, при использовании IP:
- потерянные пакеты
- пакеты не по порядку
- дубликаты пакетов
- поврежденные пакеты
TCP используется в тех приложениях, в которых требуется чтобы все пакеты были без ошибок. Например, при передаче текстовых сообщений.

## UDP User Datagram Protocol
**Ненадежный, НО быстрый**
*Без установления соединения*

Не имеет статистики, не сохраняет состояние между клиентом и сервером. Очень легкий и быстрый.
Но пакеты могут пропадать, повреждаться.

Данный протокол используется, если небольшие потери не так важны и критичны. Например, при передаче потокового видео

# Сетевой уровень
#networl_layer
**Или же интернет уровень**

Сетевой уровень отвечает за передачу пакетов из сети в сеть. А самый важный протокол это **IP**.

IP берет сегменты из транспортного протокола и добавляет метаданные, которые помогают определить, где находится клиент в локальной сети.
Также одна из функций это пересылка сообщений.

**Единица данных — Пакет**

# Канальный уровень
#data_link_layer

Канальный уровень состоит из 2 частей:
- **MAC**
- **LLC**
MAC — Media Access Control
LLC — Logical Link Layer

Здесь определяется:
- как передаются данные между системами
- как долго системы общаются
- какой объем данных может быть отправлен
- что будет если вдруг произойдет ошибка

Пакеты берутся с сетевого уровня и инкапсулируются с **добавлением новых заголовков** для MAC-адресов клиента и сервера. В конце пакета добавляется ещё одно подмножество данных, которое используется для обнаружения ошибок — то называется **хвост**.

После добавления этих метаданных данные теперь называются **кадром**.
**Единица данных — Кадр**

## Примеры протоколов канального уровня

==Ethernet== — кабельная локальная вычислительная сеть(ЛВС)
==Wi-Fi== — беспроводная ЛВС
==MAC== — адрес устройства
## Logical Link Layer
Этот уровень обеспечивает:
1. управление потоком
2. подтверждение и обработку ошибок
## Media Access Control
Этот уровень отвечает за присвоение уникального идентификационного номера, основанного на вашей сетевой карте, называемого **MAC-адресом**.

# Физический уровень
#physical_layer
**Биты могут передаваться с помощью электричества, радиоволн, света**.

Функция данного уровня заключается в преобразовании кадров в байты(8 бит) и отправить их по какому-либо способу транспортировки.

В итоге запрос будет передан на сервер, а сервер пройдет через тот же процесс, но в обратном порядке.

**Единица данных — Бит**
