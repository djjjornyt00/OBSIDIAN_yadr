==DNS== — это иерархическая и децентрализованная система именования, используемая для преобразования удобных доменных имен в IP-адреса.

# Этапы DNS-поиска
1. Клиент вводит example.com в браузере, запрос попадает в интернет и принимается *DNS-резолвером*.
2. Затем резолвер рекурсивно обращается к корневому DNS-серверу имён.
3. Корневой сервер отвечает резолверу адресом домена верхнего уровня (Top-Level Domain)
4. Затем резолвер отправляет запрос в TLD.com
5. Дальше сервер TLD отвечает IP-адресом сервера имен домена, example.com
6. Рекурсивный резолвер отправляет запрос на сервер имен домена
7. IP-адрес example.com затем возвращается резолверу из сервера имен
8. DNS-резолвер затем отвечает браузеру с IP-адресом первоначально запрошенного домена

# Типы серверов
### DNS-резолвер(resolver)
==DNS-резолвер== (также известный как DNS-рекурсивный резолвер) —  это первая точка в DNS-запросе. Он выступает посредником между ==клиентом== и ==DNS-сервером имен==.
После получения запроса от клиента резолвер либо отвечает кэшированными данными, либо  отправляет запрос корневому серверу имен → запрос к серверу имен в TLD → последний запрос к авторитативному серверу имен.
После получения ответа от авторитативного сервера имен, содержащего запрошенный айпи-адрес, рекурсивный резолвер отправляет ответ клиенту
### DNS-корневой сервер(root)
Каждому резолверу известно 13 DNS-корневых серверов имен. Существует 13 типов, но по всему миру существует несколько копий каждого из них, которые используют маршрутизацию Anycast для быстрого отклика.
### Сервер имен TLD
Данный сервер хранит информацию обо всех доменных именах, которые имеют общее расширение домена, например, или что-то, что идет после последней точки URL .com .net 
###### IANA
Управление серверами имен TLD осуществляется с помощью Internet Assigned Numbers Authority(IANA).
IANA делит серверы на 2 основные группы:
- общие домены верхнего уровня: *.com .org .net* итд
- домены верхнего уровня в странах кода: домены, специфичные для конкретных стран *.uk .us .ru .jp*

### Авторитативный DNS-сервер
Обычно последний шаг резолвера на пути к IP-адресу.
Этот сервер содержит информацию, специфичную для используемого домена (google.com) и может предоставить рекурсивному резолверу IP-адрес этого сервера.

# Типы запросов
## Рекурсивный
В рекурсивном запросе клиент требует, чтобы сервер(обычно резолвер) ответил клиенту либо запрошенной записью ресурса, либо сообщением об ошибке.

## Итеративный
В итеративном запросе клиент предоставляет имя хоста, и резолвер возвращает лучший ответ.

## Нерекурсивный
Это запрос, на который резолвер уже знает ответ. Он либо сразу возвращает запись, потому что хранит ее уже в кэше, либо сразу обращается к серверу имен, который является авторитативным для записи, то есть точно содержит айпи-адрес.
Получается нет надобности в дополнительных раундах.

# Типы записей
==DNS-записи== — это инструкции, которые находятся на авторитативных DNS-серверах и предоставляют информацию о домене, включая IP-адрес, и как обрабатывать запросы к этому домену.

Часто используемые типы записи:
- **A (Запись адреса)**: Это запись, в которой хранится IP-адрес домена.
- **AAAA (IP Version 6 Address record)**: запись, содержащая IPv6-адрес домена (в отличие от записей A, где хранится IPv4-адрес).
- **CNAME (каноническая запись имени):** пересылает один домен или поддомен на другой домен, НЕ предоставляет IP-адрес.
- **MX (запись обменчика почтой):** Направляет почту на почтовый сервер.
- **TXT (Text Record):** Эта запись позволяет администратору хранить текстовые заметки в записи. Эти записи часто используются для обеспечения безопасности электронной почты.
- **NS (записи сервера имён):** Хранит сервер имён для DNS-записи.
- **SOA (Начало полномочий):** хранит административную информацию о домене.
- **SRV (запись местоположения сервиса):** Указывает порт для конкретных сервисов.
- **PTR (запись указателя обратного поиска):** Предоставляет доменное имя при обратном поиске.
- **CERT (запись сертификата):** Хранит сертификаты с открытым ключом.

# Поддомен(Subdomain)
Поддомен — это дополнительная часть основного доменного имени. Часто используется для логического разделения сайта на разделы.

Пример:
*.blog.example.com*
*careers.example.com*

# DNS-зона
Это отдельная логическая часть пространства, которая делегируется юридическому лицу, такому как лицу, организации или компании, отвечающему за поддержание DNS-зоны.
DNS-зона является административной функцией, позволяет детально управлять DNS-компонентами, такими как авторитативные сервера имен.

# DNS-кэш
**DNS-кэш** (иногда называемый DNS-резолвером) — это временная база данных, поддерживаемая операционной системой компьютера, которая содержит записи всех недавних посещений и попыток посещения веб-сайтов и других интернет-доменов.
DNS-кэш — это просто память о последних поисках DNS

Система доменных имён реализует время жизни (TTL) для каждой DNS-записи. TTL определяет количество секунд, в течение которых запись может быть кэширована DNS-клиентом или сервером. Когда запись хранится в кэше, то TTL-значение тоже сохраняется. Сервер продолжает обновлять TTL записи, хранящейся в кэше, отсчитывая каждую секунду. Когда она достигает нуля, запись удаляется или удаляется из кэша. В этот момент, если поступает запрос к этой записи, DNS-сервер должен начать процесс разрешения.
